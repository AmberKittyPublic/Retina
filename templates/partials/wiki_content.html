What is Retina?

Retina is a full-featured Discord bot inspired by Dyno. It provides moderation, auto-moderation, logging, giveaways, tickets, leveling/XP, reaction roles, custom Lua commands, welcome/goodbye messages, AFK tracking, self-assignable roles, scheduling, and more — all configurable via the web dashboard.

Quick Start

  1. Invite the bot — Use the Invite button on the dashboard for servers that don't have the bot yet.
  2. Open the Dashboard — Visit the bot's web dashboard and login with Discord.
  3. Select a Server — Choose a server you manage from My Servers.
  4. Enable Modules — Toggle modules on/off using the switches on the server config page.
  5. Configure Auto-Mod — Set up spam, caps, links, mentions, emote, and banned word filters with custom actions (delete/warn/timeout/kick/ban).
  6. Set Up Logging — Create a channel named #mod-logs for moderation and event logs.

Moderation & Extended Moderation

Retina offers full moderation capabilities. Commands require the appropriate Discord permissions (shown in the commands page). Warning data is stored in the database and can be viewed with /warnings.

Extended features:

  • Softban — Ban a user and immediately unban to clear their recent messages
  • Voice moderation — Move, disconnect, deafen, and server-mute users in voice channels
  • Case management — View individual warning cases with /case and edit reasons with /reason
  • Mod notes — Attach notes to users with /notes add for internal reference
  • Warning management — Clear all warnings or delete individual ones

Auto-Mod Configuration

Auto-mod supports 7 rule types, each with 5 possible actions. Configure via the web dashboard under each server's settings.

Spam DetectionTriggers when a user sends N messages within M seconds
Excessive CapsTriggers when a message exceeds N% capital letters
Link DetectionBlocks messages containing URLs
Mass MentionsTriggers when a message contains N+ mentions
Emote SpamTriggers when a message contains N+ custom emotes
Banned WordsBlocks messages containing specified words or phrases
Max Message LengthBlocks messages exceeding N characters

Actions: Delete message, Warn user, Timeout (with configurable duration), Kick, or Ban.

Channel and role whitelists/blacklists let you exempt specific channels or roles from auto-mod filtering.

Logging

When enabled, Retina sends detailed logs to the #mod-logs channel. The following events are tracked:

  • Message edits and deletions
  • Member joins, leaves, and role changes
  • Channel creation and deletion
  • Voice state changes (join, leave, move)

Welcome / Goodbye

{mention}User mention (@user)

Custom Commands (Lua)

Custom commands use the ! prefix and are powered by Lua scripting. Here's the available API:

api.send(channel_id, text)Send a message to a channel
api.reply(text)Reply in the same channel
api.ban(user_id, reason)Ban a user
api.kick(user_id)Kick a user
api.timeout(user_id, minutes)Timeout/mute a user (max 40320 min)
api.warn(user_id, reason)Add a warning to a user
api.add_role(user_id, role_id)Add a role to a user
api.remove_role(user_id, role_id)Remove a role from a user
api.has_role(role_id)Check if the command author has a role
api.embed(title, description, color)Send an embed (color is decimal)
api.embed_to(channel_id, title, description, color)Send an embed to a specific channel
api.react(emoji)React to the command message
api.add_reaction(channel_id, message_id, emoji)React to any message
api.delete()Delete the command message
api.delete_message(channel_id, message_id)Delete any message
api.edit(content)Edit the bot's reply message
message.author_idAuthor's Discord ID
message.author_nameAuthor's username
message.author_mentionAuthor's mention string
message.channel_idChannel ID
message.channel_nameChannel name
message.guild_idGuild/server ID
message.guild_nameGuild/server name
message.contentFull message content
args[1], args[2], ...Command arguments

A Lua command should return a string to reply with, or use the API functions for more complex actions.

Reaction Roles

Configure reaction roles via the web dashboard. When a user reacts to a configured message with the specified emoji, they are automatically assigned the linked role. Removing the reaction removes the role.

To set up: find a message in your server, note its channel ID and message ID, then add the reaction role mapping in the dashboard with the emoji and role ID.

Giveaways

Create giveaways with /giveaway create. Users enter by reacting with 🎉. At the end of the duration, a random winner is selected. Configurable winner count and manual rerolling supported.

Giveaways persist across bot restarts — any expired giveaways are resolved on startup.

Tickets

Set up a ticket system using /ticket setup with a category ID (and optional staff role). A panel message with 🎫 reaction is created. Users react to open a private ticket channel where they can discuss with staff.

Features:

  • Staff can claim, close, and reopen tickets
  • Add or remove users from ticket channels
  • Ticket status tracked in database (open/claimed/closed)

Leveling / XP

Users earn XP by sending messages (configurable per-message XP, cooldown, and minimum character count). Level is calculated as sqrt(xp / 50).

Features:

  • View your rank and level with /rank
  • Top 10 leaderboard with /leaderboard
  • Configure XP rate, cooldown, and min chars via the web dashboard
  • Assign role rewards at specific levels
  • Admins can manually set or add XP

AFK

Set your AFK status with /afk [message]. When someone mentions you while you're AFK, they'll be notified of your status. When you return and send a message, your AFK is automatically removed.

View all AFK users with /afk-list.

Self-Assignable Roles

Allow users to pick their own roles. Admins create ranks with /addrank specifying a name and role. Users can then join or leave ranks with /rank <name> (toggles on/off).

Manager

Server management utilities:

  • Moderator Roles — Designate specific roles as moderators (/addmod, /delmod, /listmods)
  • Nickname Management — Change or reset user nicknames (/nick)
  • Role Assignment — Add or remove roles from users (/addrole, /delrole)

Scheduling

Schedule moderation actions and announcements:

  • Temp-ban — Ban a user for a set duration, automatically unbanned afterward
  • Temp-mute — Mute a user for a set duration
  • Scheduled announcements — Send one-time or repeating announcements to a channel at a specified date/time
