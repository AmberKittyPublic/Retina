What is Retina?

Retina is a full-featured Discord bot inspired by Dyno. It provides moderation, auto-moderation, logging, giveaways, tickets, leveling/XP, reaction roles, custom Lua commands, welcome/goodbye messages, and scheduling — all configurable via the web dashboard.

Quick Start

  1. Invite the bot — Use the Invite button on the dashboard for servers that don't have the bot yet.
  2. Open the Dashboard — Visit the bot's web dashboard and login with Discord.
  3. Select a Server — Choose a server you manage from My Servers.
  4. Enable Modules — Toggle modules on/off using the switches on the server config page.
  5. Configure Auto-Mod — Set up spam, caps, links, mentions, emote, and banned word filters with custom actions (delete/warn/timeout/kick/ban).
  6. Set Up Logging — Create a channel named #mod-logs for moderation and event logs.

Modules Overview

ModerationBan, kick, warn, mute (timeout), purge, slowmode, lockdown
Auto-ModAutomatic message filtering: spam, excessive caps, links, mass mentions, emote spam, banned words, max length
LoggingLogs message edits/deletes, member join/leave, channel changes, voice state updates to #mod-logs
Welcome / GoodbyeCustomizable join/leave messages with {user}, {mention}, {guild} placeholders
Custom CommandsLua-scripted commands triggered by !commandname. Full Discord API bindings available.
Reaction RolesAssign and remove roles when users react to a message
GiveawaysCreate giveaways with reaction entry, random winner selection, and scheduled end times
TicketsReaction-based ticket system with private channels, claim/close/add/remove
Leveling / XPEarn XP by chatting, level up, and receive role rewards
SchedulingTemp-ban, temp-mute, and timed announcements (one-time or repeating)

Custom Commands (Lua)

Custom commands use the ! prefix and are powered by Lua scripting. Here's the available API:

api.send(channel_id, text)Send a message to a channel
api.reply(text)Reply in the same channel
api.ban(user_id, reason)Ban a user
api.kick(user_id)Kick a user
api.timeout(user_id, minutes)Timeout/mute a user (max 40320 min)
api.warn(user_id, reason)Add a warning to a user
api.add_role(user_id, role_id)Add a role to a user
api.remove_role(user_id, role_id)Remove a role from a user
api.has_role(role_id)Check if the command author has a role (returns true/false)
api.embed(title, description, color)Send an embed to the same channel (color is decimal)
api.embed_to(channel_id, title, description, color)Send an embed to a specific channel
api.react(emoji)React to the command message (unicode or <:name:id>)
api.add_reaction(channel_id, message_id, emoji)React to any message
api.delete()Delete the command message
api.delete_message(channel_id, message_id)Delete any message
api.edit(content)Edit the bot's reply message content
message.author_idAuthor's Discord ID
message.author_nameAuthor's username
message.author_mentionAuthor's mention string (<@id>)
message.channel_idChannel ID
message.channel_nameChannel name
message.guild_idGuild/server ID
message.guild_nameGuild/server name
message.contentFull message content
args[1], args[2], ...Command arguments

A Lua command should return a string to reply with, or use the API functions for more complex actions.
